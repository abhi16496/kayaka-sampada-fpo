Create a modern professional Registration Management PWA with Admin Approval System.

Tech Stack:

* Base framework: Progressive Web App (PWA)
* Mobile-first responsive design
* Works perfectly on Web + Android + iPhone screen ratios
* Clean enterprise-level UI/UX
* Fast loading and smooth animations
* Modern card-based layout
* Glassmorphism + subtle gradients + minimal professional design
* Dark mode and light mode support
* Fully responsive admin dashboard
* Production-ready scalable architecture

Main Features:

1. User Registration Module
   Create a beautiful registration form with validation and smooth UI transitions.

Required fields:

* Full Name
* Father / Mother / Husband Name
* Full Address
* Area / Village
* PIN Code
* Taluk
* District
* Phone Number
* Email ID

Additional Features:

* Form validation
* Auto uppercase support for names and address
* Mobile number validation
* Email validation
* Loading animation during submit
* Success popup after registration
* Save registration timestamp
* Generate unique registration ID automatically
* Prevent duplicate mobile number registration

2. Registration Workflow
   After user submits form:

* Data should go into "Pending Approval" state
* Admin should receive the request in Admin Panel
* User should see message:
  "Your registration is submitted successfully and waiting for admin approval."

Admin can:

* Approve registration
* Reject registration
* View complete applicant details
* Search registrations
* Filter by status
* View registration date and time

3. Admin Panel
   Design a professional admin dashboard with:

* Sidebar navigation
* Dashboard analytics cards
* Total Registrations counter
* Pending Requests counter
* Approved Users counter
* Rejected Users counter

Admin Sections:

* Pending Applications
* Approved Registrations
* Rejected Registrations
* Search & Filter
* Export data to Excel/PDF
* User Details View Modal
* Mobile responsive admin dashboard

Approval Flow:

* Pending registrations move to Approved list after admin approval
* Approved data should be stored in separate approved database/table
* Rejected registrations should move to rejected list
* Real-time status updates

4. Database Structure
   Create scalable backend architecture with:

* Users Collection/Table
* Pending Registrations
* Approved Registrations
* Rejected Registrations
* Admin Authentication

Use production-ready database structure with proper indexing.
